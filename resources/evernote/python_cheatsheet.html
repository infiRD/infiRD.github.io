- clear all as in Matlab [60]: 
Python has different philosophy for variable management than Matlab. Python uses the notion of namespaces and local variables in functions to prevent the problem of global namespace polution, therefore command clear all is not neccesary

To obtain identical result as with clear all command, one could use:
   - clear all variables (input directly into iPython shell):
reset -f
   - or in code:
from IPython import get_ipython
get_ipython().magic('%reset -f')

   - this approach works, but causes problems in Spyder IDE, which relies on the state of ipython/python
     session. It would be unable to execute: namespace.pop('__file__')
   - Spyder has inbuilt feature to do this: Spyder -> Preferences -> Run -> Clean all variables before execution
